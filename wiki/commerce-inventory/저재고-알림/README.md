# 저재고 알림

> 작성일: 2026-05-13 | 수정일: 2026-05-13 | 유형: 정책 | 관련 레포: example-inventory-worker

## 개요

재고가 임계값 미만으로 떨어지면 셀러에게 알려, 셀러가 매출 기회를 놓치거나 구매자가 품절로 불편을 겪는 일을 줄입니다. 알림 발행은 재고 도메인이 담당하고, 실제 전송 채널(이메일·푸시)은 별도 알림 서비스에 위임합니다.

## 임계값 정책

| 단계 | 기본값 | 사용자 설정 가능 |
|------|--------|----------------|
| 기본 임계값 | 5개 | ❌ (코드 기본값) |
| 셀러별 임계값 | — | ✅ 셀러가 상품별로 지정 가능 |
| 카테고리별 권장 임계값 | 카테고리별 평균 매출량의 3일치 | 분석 결과 기반 추천 |

셀러가 별도 설정하지 않으면 기본값 5개가 적용됩니다. 카테고리별 권장은 운영 분석 결과로 셀러에게 추천하지만 강제는 아닙니다.

## 트리거 시점

저재고 상태 진입은 `stock-fulfillment`가 재고를 차감한 직후 검사합니다:

```
stock-fulfillment 처리 후:
   if (stock.available < threshold) and (이전 상태가 정상이었다면):
       low-stock-alert.trigger(product_id, current_available)
```

이미 저재고 상태인 상품에 대해서는 재차 알림을 보내지 않습니다(스팸 방지).

저재고에서 복귀하는 경우(입고 등)는 별도 이벤트로 발행되며, 알림 서비스가 "재입고됐어요" 알림을 보낼지 결정합니다.

## 알림 이벤트 스키마

`commerce.inventory.low_stock`:

```json
{
  "event_id": "uuid",
  "event_type": "entered" | "recovered",
  "seller_id": 999,
  "product_id": 12345,
  "current_available": 3,
  "threshold": 5,
  "occurred_at": "2026-05-13T10:00:00Z"
}
```

다운스트림(알림 서비스):
- 셀러 푸시 알림 / 이메일
- 셀러 대시보드의 알림 뱃지
- 마케팅: 인기 상품이 저재고면 푸시 광고 차단 (재고 부족 시 광고하면 사용자 불만 발생)

## 알림 채널 분리 이유

재고 도메인은 "저재고 상태가 됐다"는 사실만 발행합니다. 채널(어떻게 알릴지)·빈도(몇 번 알릴지)·수신자(셀러 본인인지 그 회사 직원인지)는 **알림 서비스의 책임**입니다.

이렇게 분리하지 않으면:
- 알림 채널이 추가될 때마다 재고 도메인 코드 변경 필요
- 셀러별 알림 설정(시간대, 채널 선호도)을 재고 도메인이 관리해야 함

도메인 경계를 분명히 두기 위해 발행만 하고 끝냅니다.

## 중복 알림 방지

`stock-fulfillment`가 매번 트리거하면 한 상품에 대해 중복 이벤트가 발생할 수 있습니다(특히 같은 시점에 여러 주문이 처리될 때). 방지 메커니즘:

1. Redis에 `low_stock:notified:{product_id}` 키, TTL 1시간
2. 첫 진입 시 키 설정, 이미 키가 있으면 이벤트 발행 skip
3. 입고로 복귀 시 키 삭제 → 다음 저재고 진입 시 다시 알림 가능

## 임계값 변경의 영향

셀러가 임계값을 변경하면 그 즉시 알림 조건이 바뀝니다. 변경 직후 현재 재고가 이미 새 임계값 미만이면 즉시 알림 발행됩니다.

임계값이 너무 높게 설정되면 매일 같은 상품에 대해 알림이 발생할 수 있습니다. 셀러 대시보드에서 "최근 7일간 알림 빈도가 많은 상품"을 표시해 임계값 재검토를 유도합니다(추후 작업).

## 의사결정 배경

- **왜 발행만 하고 채널은 분리하는가**: 알림 채널의 변경 빈도가 재고 로직보다 훨씬 높습니다. (이메일 → 푸시 → 슬랙 봇 등). 채널을 도메인 안에 두면 알림 인프라 변경이 도메인 코드 변경을 강제합니다. Kafka 이벤트로 발행만 하고 알림 도메인이 처리합니다.
- **왜 1시간 동안 중복 알림을 막는가**: 너무 짧으면 같은 정보가 반복 전송되어 셀러가 알림을 무시하기 시작합니다. 너무 길면 다른 사용자 구매로 재고가 더 줄어든 상황을 놓칩니다. 1시간이 절충점입니다.
- **왜 카테고리별 임계값을 강제하지 않는가**: 권장값과 실제 셀러 운영 패턴이 다를 수 있습니다(소량 다품종 셀러 vs 대량 단일품 셀러). 추천만 하고 결정은 셀러에게 맡깁니다.
