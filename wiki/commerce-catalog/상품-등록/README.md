# 상품 등록

> 작성일: 2026-05-13 | 수정일: 2026-05-13 | 유형: 정책 | 관련 레포: example-catalog-api

## 개요

셀러가 입력한 상품 정보를 검수해 카탈로그에 정식 등록하는 과정을 다룹니다. 핵심 책임은 (1) 필수 정보 누락 방지, (2) 자동 검수 룰로 품질 보장, (3) 다운스트림(검색·재고) 전파를 위한 이벤트 발행입니다.

## 등록 시나리오

1. 셀러가 `POST /products`로 상품 데이터를 입력합니다.
2. 입력 데이터는 즉시 임시 저장(상태: `pending`)되고, 동기 검수 룰이 적용됩니다.
3. 검수 통과 시 상태가 `active`로 전환되고 노출이 시작됩니다.
4. 검수 실패 시 상태 `rejected`와 함께 사유 코드가 셀러에게 전달됩니다.
5. 등록 완료(`active` 전환) 시 `commerce.catalog.product.changed` 이벤트가 발행됩니다.

## 필수 필드

| 필드 | 검증 | 비고 |
|------|------|------|
| `name` | 1~80자, 금칙어 미포함 | 노출명. 셀러 브랜드명 포함 가능 |
| `category_ids` | 1개 이상, 최대 2개 | 리프 노드만 허용 (카테고리-구조 참조) |
| `seller_id` | 활성 셀러 | 비활성 셀러는 등록 차단 |
| `images[]` | 최소 1장, 최대 10장 | 대표 이미지는 첫 번째 |
| `prices[]` | 최소 1개 (정가) | 0원 이하 금지 |

## 자동 검수 룰

1. **금칙어 차단** — 상품명·설명에 [금칙어 사전](../금칙어-사전.md)에 등록된 단어가 있으면 자동 거절. Redis에 캐시.
2. **카테고리-키워드 일치도** — 상품명 키워드와 매핑된 카테고리가 합리적인지 NLP 모델로 점검(임계값 0.6 미만 → 수동 검수로 전환).
3. **이미지 사이즈/포맷** — JPEG/PNG, 최소 800x800, 최대 10MB.
4. **가격 적정선** — 동일 카테고리 중앙값 대비 1/10 미만 또는 10배 초과 시 수동 검수.
5. **셀러 신뢰 등급** — 신규 셀러의 첫 5건은 자동 통과 차단, 무조건 수동 검수.

## 이벤트 스키마

`commerce.catalog.product.changed` 토픽 메시지:

```json
{
  "event_id": "uuid",
  "event_type": "created" | "updated" | "deleted",
  "product_id": 12345,
  "seller_id": 999,
  "category_ids": [100, 102],
  "occurred_at": "2026-05-13T10:00:00Z"
}
```

본문은 페이로드에 싣지 않고, 컨슈머가 필요 시 MySQL에서 최신 본문을 로드합니다. 메시지 사이즈를 작게 유지하고 정합성을 한 곳(MySQL)에서 보장하기 위한 결정입니다.

## 의사결정 배경

- **왜 동기 검수 + 비동기 인덱싱인가**: 검수 결과는 즉시 셀러에게 알려야 사용자 경험이 좋지만, 검색 인덱싱은 5초 정도 지연되어도 무방합니다. 두 책임을 분리해 등록 응답 지연을 줄였습니다.
- **왜 이벤트에 본문을 싣지 않는가**: 본문이 큰 상품(이미지 메타, 다국어 설명)은 메시지가 수십 KB를 넘습니다. 다운스트림이 본문을 직접 필요로 하는 경우가 적고(있어도 MySQL 조회로 충분), 메시지 사이즈는 작게 두는 게 토픽 비용·재처리 부담 측면에서 유리합니다.
