# 상품 카탈로그 (commerce-catalog)

> 유형: 정책 | 관련 레포: example-catalog-api, example-catalog-worker

## 배경

쇼핑 서비스에서 셀러가 입력한 상품 데이터를 표준 카탈로그로 만들어, 구매자가 검색·탐색할 수 있도록 노출하는 도메인입니다. 셀러는 자기 기준의 정보를 입력하지만, 구매자는 통합된 카탈로그 뷰를 기대합니다. 그 사이에서 데이터를 검수·정규화·전파하는 일이 이 도메인의 책임입니다.

## 풀려는 문제

- **카탈로그 품질 차이**: 셀러가 자유 입력하면 카테고리 매핑, 이미지 품질, 가격 표기 방식이 셀러마다 다릅니다. 검수와 정규화 룰이 필요합니다.
- **검색 노출 지연**: 상품을 등록하거나 가격을 바꿔도 검색 인덱스에 반영되지 않으면 구매자에게 보이지 않습니다. MySQL 변경을 Elasticsearch까지 자동 전파해야 합니다.
- **가격 정책의 복잡성**: 정가, 시즌 할인, 쿠폰 할인이 동시에 적용되면 어떤 가격을 노출할지 결정해야 합니다.
- **이미지 운영 비용**: 상품 이미지는 트래픽이 크고 리사이즈가 빈번해 CDN 없이는 운영이 어렵습니다.

## 사용자

| 사용자 | 어떻게 사용하는가 |
|--------|--------------------|
| **셀러** | 상품 등록 API로 상품을 입력하고, 등록 후 검수 결과 알림을 받는다. |
| **구매자** | 검색·카테고리 탐색 결과로 노출된 상품을 본다. 직접 이 도메인과 상호작용하지는 않는다. |
| **MD/운영팀** | 검수 룰, 카테고리 트리, 노출 우선순위를 정의·관리한다. |
| **타 도메인 (commerce-order, commerce-inventory)** | 상품 ID와 카탈로그 정보를 조회한다. 카탈로그가 단일 진실의 원천. |

## 성공 기준

- 상품 등록 후 검색에 노출되기까지 **5초 이내** (p95)
- 검수 통과율 **95% 이상** (자동 룰로 통과 가능한 상품 비율)
- 검색 인덱스와 MySQL의 데이터 불일치 **0건** (자동 동기화)
- 이미지 CDN 캐시 적중률 **90% 이상**
