# Glossary — 상품 카탈로그

> 이 도메인에서 자주 쓰이는 용어를 풀어 정리합니다.

## 데이터/엔티티

**상품 (product)**
카탈로그의 핵심 데이터 단위. 한 셀러가 등록한 하나의 판매 단위에 대응합니다. 이름, 설명, 노출 상태, 등록일 등의 기본 정보를 가지며, 이미지·가격·카테고리는 별도 데이터로 분리되어 연결됩니다.

**카테고리 (category)**
상품을 분류하는 트리 구조. "여성/의류/원피스" 같은 경로로 표현됩니다. 깊이는 최대 3단계까지 허용하며, 상품은 리프 노드(가장 아래 단계)에만 매핑할 수 있습니다.

**상품 이미지 (product-image)**
상품에 첨부되는 이미지의 메타데이터. 파일 경로, 크기, 노출 순서를 가지며 원본 파일은 S3 비공개 버킷에 저장됩니다. 외부에는 CloudFront를 통해 노출됩니다.

**가격 (price)**
한 상품의 정가, 할인가, 할인 적용 기간을 보관합니다. 한 상품은 여러 가격 정책을 동시에 가질 수 있고, 노출 시점에 우선순위 규칙으로 최종 가격이 계산됩니다.

**검색 인덱스 문서 (search-index-document)**
Elasticsearch의 단일 도큐먼트. 상품 기본 정보 + 카테고리 경로 + 노출 가격 + 검색 키워드 + 가중치 신호를 정규화하여 담습니다.

## 프로세스/이벤트

**상품 등록 (catalog-registration)**
셀러의 상품 입력을 검수해 카탈로그에 정식 등록하는 파이프라인. 이미지 업로드, 검수 룰 적용, 데이터 저장, Kafka 이벤트 발행을 한 단위로 묶어 처리합니다.

**검색 인덱스 동기화 (search-index-sync)**
MySQL의 상품 변경을 Elasticsearch에 반영하는 비동기 컨슈머. Kafka 이벤트를 구독해 최신 본문을 로드하고 검색 문서를 갱신합니다.

**가격 변경 파이프라인 (price-update-pipeline)**
단건 또는 일괄 가격 변경을 처리하는 파이프라인. 변경 후 검색 노출 가격도 즉시 반영하기 위해 search-index-sync를 트리거합니다.

**검수 (validation)**
상품 등록 시 자동으로 적용되는 룰 묶음. 필수 필드 누락, 금칙어, 카테고리-상품 불일치 등을 검출합니다.

**노출 우선순위 (display rank)**
검색 결과에서 상품이 노출되는 순서를 결정하는 점수. 등록일, 가격, 셀러 평점, 재고 가용성 등이 가중치로 작용합니다.

## 인프라/외부

**CDN (CloudFront)**
상품 이미지를 전 세계 엣지에서 캐시하여 빠르게 제공하는 콘텐츠 전송 네트워크. 원본은 S3 비공개 버킷이며, 외부 트래픽은 모두 CloudFront를 거칩니다.

**이벤트 토픽**
Kafka의 메시지 채널. 이 도메인에서는 `commerce.catalog.product.changed`를 사용합니다.

## 자주 혼동되는 개념

| 비슷해 보이지만 다른 것 | 차이 |
|-------------------------|------|
| `product` vs `product-image` | 상품 본문 vs 그에 첨부되는 이미지 메타. 이미지는 상품에 종속(owns). |
| `price` vs 노출 가격 | DB에 저장된 가격 정책의 모음 vs 우선순위 계산 후 결정된 최종 가격. |
| `category` 매핑 vs 노출 카테고리 | 상품에 부여된 카테고리(최대 2) vs 검색 인덱스에 펼쳐진 경로 전체. |
