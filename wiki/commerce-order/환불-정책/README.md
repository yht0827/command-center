# 환불 정책

> 작성일: 2026-05-13 | 수정일: 2026-05-13 | 유형: 정책 | 관련 레포: example-order-api

## 개요

결제 완료된 주문을 사용자가 취소하거나 CS팀이 보정해야 할 때, 결제 수단별로 다른 환불 절차를 일관된 인터페이스로 처리합니다. 전체 환불뿐 아니라 라인 단위 부분 환불도 다룹니다.

## 환불 가능 조건

| 단계 | 환불 가능 여부 |
|------|----------------|
| 결제 완료 (`paid`) ~ 배송 시작 전 | 사용자 자기 환불 가능 |
| 배송 중 | 자기 환불 불가, CS 승인 필요 |
| 배송 완료 ~ 수령 후 7일 | 단순 변심 환불 가능 (배송비 사용자 부담) |
| 수령 후 7일 이후 | 단순 변심 환불 불가, 하자 사유만 가능 (CS 검토) |
| 디지털 상품, 사용 시작 후 | 환불 불가 (특별 사유 제외) |

## 환불 흐름

1. 사용자/CS가 환불 요청 (`POST /orders/{id}/refunds`)
2. 환불 가능 조건 검증
3. 환불 금액 계산:
   - 라인 환불 금액 = `unit_price * quantity`
   - 쿠폰 회수: 환불 라인에 적용된 할인은 같이 회수
   - 배송비: 전체 환불 시 환불, 부분 환불은 사용자 부담
4. PG 환불 API 호출 (결제 수단별 어댑터)
5. `payment.refunded` 또는 `partial_refunded` 상태 전이
6. `order.refunded` 또는 `partial_refunded` 상태 전이
7. `commerce.order.refunded` 이벤트 발행 → 재고 도메인이 구독해 재고 복원

## 결제 수단별 환불

| 수단 | 절차 | 소요 시간 |
|------|------|----------|
| 신용카드 | PG에 즉시 환불 요청, 카드사 처리 | 영업일 기준 3~5일 |
| 계좌이체 | PG에 환불 요청, 사용자 계좌로 송금 | 영업일 기준 1~3일 |
| 간편결제 | 각 간편결제사 API | 즉시 ~ 1일 |
| 포인트 | 내부 시스템에서 즉시 적립 | 즉시 |

분할 결제(카드 + 포인트)의 부분 환불 시, 환불 금액을 어떤 수단에서 차감할지 결정해야 합니다. 기본 정책: **외부 결제(카드) 우선 환불 → 잔여를 포인트에서 차감**. 사용자가 환불 시 수단을 지정할 수 있는 옵션은 별도 검토 대상(미구현).

## 부분 환불

라인 단위 부분 환불 시:

- 환불되지 않는 라인의 결제 상태는 유지
- 주문 상태는 `partial_refunded`로 전환
- 모든 라인이 환불되면 자동으로 `refunded`로 전환

쿠폰이 주문 단위로 적용된 경우(예: "전체 주문 10% 할인"), 부분 환불 시 쿠폰 할인 금액을 비례 회수합니다. 라인 단위 쿠폰(예: "특정 상품 5,000원 할인")은 그 라인의 환불 시에만 회수합니다.

## 환불 거부 사유

| 코드 | 사유 |
|------|------|
| `BEYOND_PERIOD` | 환불 가능 기간 초과 |
| `ALREADY_REFUNDED` | 이미 환불 완료된 주문 |
| `DIGITAL_USED` | 디지털 상품 사용 시작 후 |
| `PG_REJECTED` | PG가 환불 거부 (만료된 카드 등) |
| `MANUAL_REVIEW_REQUIRED` | 자동 처리 불가, CS 수동 검토 필요 |

## 의사결정 배경

- **왜 쿠폰을 환불 시 회수하는가**: 쿠폰을 적용해 할인받은 금액만큼 PG에서 적게 청구했기 때문에, 환불 금액도 청구된 금액 기준으로 계산해야 정합성이 맞습니다. 회수하지 않으면 사용자가 할인받은 만큼을 추가로 돌려받는 셈이 됩니다.
- **왜 외부 결제 수단을 먼저 환불하는가**: 포인트는 내부에서 즉시 처리되어 사용자에게 빠르게 보입니다. 외부 결제는 시간이 걸리므로 먼저 처리해 사용자 대기 시간을 줄이고, 정산상으로도 외부 거래를 우선 정리하는 게 깔끔합니다.
- **왜 배송 시작 후 자기 환불을 막는가**: 배송 중인 상품은 재고 복원과 정산 처리가 복잡해집니다. CS가 운송사와 협의해 회수 절차를 진행해야 하므로 자동 환불 흐름에서 분리합니다.
