`wiki/` 문서의 생애주기를 점검하고 정리 후보를 리포트하는 스킬.

---

## 인자

- `ARGS[0]` (선택): 점검 범위. 생략 시 전체 도메인 점검.
  - 도메인: `/domain-cleanup asset-factory`
  - 도메인/주제: `/domain-cleanup asset-factory/팩토리-파이프라인`

## 점검 항목

### 1. status.md 오래된 완료 항목

- 각 도메인의 `status.md`에서 ✅ 항목을 찾는다
- 해당 행이 마지막으로 수정된 시점을 `git log`로 확인한다
- **20일 이상 경과**한 ✅ 항목을 정리 후보로 리포트한다
- 판정 기준: `git log -1 --format=%aI -S "{항목 ID}" -- wiki/{도메인}/status.md`로 해당 ID가 마지막으로 변경된 날짜를 추출

### 2. 완료된 일회성 작업 주제

- 각 도메인의 주제 폴더 README.md 헤더에서 `유형: 작업`이고 `상태: 완료`인 문서를 찾는다
- 해당 문서를 정리 후보로 리포트한다

### 3. 장기 미수정 문서

- 각 주제 폴더의 README.md에 대해 `git log -1 --format=%aI`로 마지막 수정일을 확인한다
- **60일 이상 미수정**인 문서를 리포트한다 (참고 정보, 즉시 정리 대상은 아님)

### 4. 종료된 도메인 감지

- 도메인 README.md 헤더에 `상태: 종료`가 있는 도메인을 리포트한다

### 5. 용어 불일치

- 도메인 `glossary.md`에 정의된 용어와 주제 문서 본문에서 사용하는 표현을 비교한다
- glossary에 정의된 용어의 다른 표기(예: "원장" vs "레저")가 문서에서 쓰이고 있으면 리포트한다

### 6. 참조 무결성

- `architecture.md`에 링크된 주제 폴더(`{주제}/README.md`)가 실제로 존재하는지 확인한다
- 링크는 있지만 폴더/파일이 없으면 리포트한다
- 반대로, 주제 폴더는 있지만 `architecture.md`에 링크가 없는 경우도 리포트한다

### 7. ontology repos 정합성

- `ontology/index.yaml`의 각 도메인 `repos` 필드에 매핑된 레포가 `projects/` 디렉토리에 실제로 존재하는지 확인한다
- 매핑은 있지만 디렉토리가 없으면 리포트한다 (clone 필요 안내)
- 도메인이 삭제/종료되면 ontology index.yaml과 abox 파일도 정리 대상으로 리포트한다

## 실행 흐름

```
1. 대상 도메인 결정 (ARGS[0] 또는 전체)
2. 도메인별 점검 실행 (1~4)
3. 리포트 출력
4. 사용자에게 정리 실행 여부 확인 (AskUserQuestion)
5. 승인된 항목만 정리 실행
```

## 리포트 형식

```markdown
# 도메인 정리 리포트

## {도메인}

### status.md 정리 대상 (20일+ 경과 완료 항목)

|----|---------|--------|--------|
| FR-1 | ... | 2026-01-15 | 55일 |

### 완료된 일회성 작업
| 주제 | 완료 표시일 |
|------|-----------|
| ... | ... |

### 장기 미수정 문서 (참고)
| 주제 | 마지막 수정 | 경과일 |
|------|-----------|--------|
| ... | ... | ... |

### 용어 불일치
| glossary 용어 | 문서 내 표현 | 파일 |
|--------------|-------------|------|
| 원장 | 레저 | 팩토리-파이프라인/README.md |

### 참조 무결성
| 문제 | 대상 |
|------|------|
| 링크 있지만 폴더 없음 | architecture.md → 주제-X/ |
| 폴더 있지만 링크 없음 | 주제-Y/ |

### ontology repos 정합성
| 레포 | 상태 |
|------|------|
| asset-factory-api | projects/에 없음 → `/sync-projects` 필요 |

```

## 정리 실행

사용자가 승인하면:

- **status.md 항목**: 해당 행을 삭제. 섹션 내 모든 행이 삭제되면 섹션 헤더도 제거
- **완료된 작업 주제**: 주제 폴더를 삭제하고, `architecture.md`와 `README.md`의 인덱스에서 해당 항목 제거
- 변경사항은 커밋하지 않는다 — 사용자가 `/commit`으로 별도 커밋

## 제약

- **읽기 우선**: 점검 단계에서는 파일을 수정하지 않는다
- **사용자 승인 필수**: 정리 실행은 반드시 사용자 확인 후에만
- 정리 시 `wiki-docs.md`의 문서 작성 규칙을 준수한다
