# Impact Phase: 영향도 분석 (복잡도 + 리스크 병렬)

`SUMMARIES_FILE`(LENS_SUMMARIES 저장 파일)을 기반으로 architect(복잡도)와 security-auditor(리스크)를 **병렬** Task로 호출한다.

## IDEA 안전 처리

IDEA_CONTEXT.idea와 clarifications를 에이전트 프롬프트에 삽입할 때, SKILL.md 공유 규칙의 "사용자 입력 안전 처리" 5단계를 적용한다.

## 1. 병렬 Task 호출

하나의 메시지에서 두 `Task()` 호출을 **동시에** 발행한다.

## 2. 결과 수집

두 Task가 모두 완료되면 결과를 수집한다:
- Architect Task 결과 → `ARCHITECT_ANALYSIS`
- ZT Task 결과 → `ZT_ANALYSIS`

### 실패 처리

- **둘 다 성공**: 정상 진행.
- **하나만 실패**: 성공한 분석 결과만으로 보고서를 작성한다. 실패한 분석은 보고서에 "분석 실패"로 표시.
- **둘 다 실패**: "분석에 실패했습니다. 다시 시도해주세요." 출력 후 중단.
