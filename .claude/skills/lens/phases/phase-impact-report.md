# Impact-Report Phase: 영향도 보고서 합성

SUMMARIES_FILE, ARCHITECT_ANALYSIS, ZT_ANALYSIS를 **간결한 PO 친화적 보고서**로 합성한다. 오케스트레이터가 직접 수행한다.

먼저 `SUMMARIES_FILE`을 Read하여 LENS_SUMMARIES 내용을 확인한다.

## 1. 보고서 합성 규칙

### 언어 원칙
- **비즈니스 언어**로 작성한다. 파일 경로 대신 **비즈니스 영역** 단위로 그룹핑한다.
- 기술 용어가 불가피하면 괄호 안에 비즈니스 용어를 병기한다.
- PO가 "그래서 어떻게 하면 돼?"에 바로 답할 수 있는 수준으로 작성한다.

### 구조 원칙
- **간결함 우선**. 보고서 전체가 화면 1~2페이지를 넘지 않도록 한다.
- 각 섹션은 **결론 먼저, 근거 다음** 순서로 작성한다.
- Report Phase에서 이미 상세 보고된 내용을 반복하지 않는다. as-is는 1~2줄 축약.
- ARCHITECT_ANALYSIS이 없으면 복잡도를 "분석 실패"로 표시한다.
- ZT_ANALYSIS가 없으면 리스크를 "분석 실패"로 표시한다.

## 2. 보고서 포맷

```markdown
## 영향도 분석: <IDEA_CONTEXT.idea 요약 (15자 이내)>

> 분석 대상: <CANDIDATES 각 name 나열>

### 현황 요약

Report Phase 보고서의 핵심 발견을 **1~2줄로 축약**.

### 요약

| 항목 | 내용 |
|------|------|
| 복잡도 | <상/중/하> — <1문장 근거> |
| 변경 범위 | <N>개 파일, <M>개 모듈 |
| DB/API 영향 | <DB 변경 필요/불필요>, <하위호환성 유지/깨짐> |
| 공수 | <예상 구현 단계 수와 핵심 작업 1줄 요약> |
| 리스크 | CRITICAL <n> / HIGH <n> / MEDIUM <n> |
| 판정 | **<바로 진행 / 추가 검토 후 진행 / 신중히 판단 필요>** |

<다중 프로젝트일 때만>
### 프로젝트 간 영향

- **배포 순서**: <의존 관계에 따른 순서, 없으면 "독립적">
- **공유 인터페이스**: <변경되는 API/모델 설명, 없으면 "없음">
</다중 프로젝트일 때만>

### 주요 리스크

<CRITICAL/HIGH만 나열. MEDIUM은 생략하거나 "외 N건"으로 표기>

<CRITICAL이 있으면>
> **즉시 확인 필요**

- <비즈니스 언어로 위험 설명> — <권고>
</CRITICAL>

<HIGH가 있으면>
**배포 전 확인:**
- <비즈니스 언어로 위험 설명> — <권고>
</HIGH>

<MEDIUM이 있으면>
> 참고: 외 <N>건의 중간 리스크 식별됨
</MEDIUM>

<리스크가 0건이면>
> 현재 코드 기준으로 특별한 리스크가 식별되지 않았습니다.
</리스크 0건>

### 체크리스트

놓치기 쉬운 시나리오를 **상위 3~5개**만 선별.

- [ ] <시나리오> — <왜 중요한지>
- [ ] ...

### 예상 구현 순서

1. <단계> — <설명>
2. ...

<다중 프로젝트이고 배포 순서 의존성이 있으면>
> 배포 순서: <어떤 프로젝트를 먼저 배포해야 하는지>
</다중 프로젝트>
```

## 3. 판정 로직

다음 기준으로 최종 판정을 결정한다:

| 조건 | 판정 |
|------|------|
| 복잡도 하 + CRITICAL 0건 + HIGH 0건 | **바로 진행** |
| 복잡도 중 + CRITICAL 0건 | **추가 검토 후 진행** |
| 복잡도 상 또는 CRITICAL 1건 이상 | **신중히 판단 필요** |
| DB 변경 필요 또는 하위호환성 깨짐 | 한 단계 상향 (바로→추가, 추가→신중) |
| 다중 프로젝트 + 프로젝트 간 공유 인터페이스 변경 | 한 단계 상향 |
| 복잡도 분석 실패 | 복잡도를 "중"으로 가정하고 판정. "(복잡도 분석 미완료)" 병기 |
| 리스크 분석 실패 | CRITICAL/HIGH를 0으로 가정하고 판정. "(리스크 분석 미완료)" 병기 |

**상향 규칙**: 상향 조건은 **중첩 적용**한다. 2개 조건이 동시에 해당되면 2단계 상향한다. 최대 상한은 "신중히 판단 필요"이다.

## 4. 보고서 출력

합성된 보고서를 **대화에 직접 출력**한다. 파일로 저장하지 않는다.

SESSION_DIR을 `realpath`로 정규화한 후, 비어있지 않고 `.lens/` 하위인지 확인한 후에만 `Bash(rm -rf <SESSION_DIR>)` — 세션 디렉토리 정리.

출력 후 안내:

```
추가로 확인하고 싶은 정책이 있으면 `/lens <쿼리>`로 다시 실행해주세요.
특정 프로젝트만 보려면 `/lens <프로젝트명>에서 <쿼리>`로 지정할 수 있습니다.
```
